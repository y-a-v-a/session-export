🤖 session-export

A single-file, offline HTML exporter for coding-agent sessions — now spanning Claude Code and OpenAI Codex. This page summarizes the work done across the session and the resulting architecture.

Claude Code OpenAI Codex MIT no deps · Node 18+ no tracking

What it does

Reads a session's .jsonl transcript straight off disk and renders it as one self-contained .html file — messages, thinking/reasoning, every tool call with its result, and subagent transcripts inline. No network, no CDN, no base64; the JSON payload is embedded and a small client script renders it with a searchable sidebar, themes, and collapsible blocks. A secret-redaction pass scrubs credential shapes before writing.

Architecture

The key design move: one format-agnostic renderer. Both exporters normalize their very different transcripts into the same Claude-shaped payload ({ header, entries[], subagents }), so the entire renderer, redaction pass, themes, and chrome are shared.

Codex support — the format gap

Codex rollouts are OpenAI Responses items, mapped onto the shared payload:

Codex record→ payload block
session_meta + turn_contextheader (id, cwd, model, tokens)
message (user / assistant)text turn
function_call / _outputtool_use / tool_result (paired by call_id)
reasoningcollapsed "reasoning hidden" note (see below)
developer role messagefoldable "instructions" context card
<subagent_notification>re-attributed from "user" → assistant turn

🔒 Finding: Codex encrypts its reasoning

Every Codex reasoning item carries an empty summary and an encrypted_content blob — and that blob is a Fernet token (decoded: leading version byte 0x80 plus an embedded timestamp matching the record). Fernet = AES-128-CBC + HMAC-SHA256 under a key held server-side by OpenAI.

So the chain-of-thought is genuinely unrecoverable from the rollout — not a limitation of the exporter. The export records that reasoning happened (N steps), never what. Contrast: Claude stores thinking in cleartext.
